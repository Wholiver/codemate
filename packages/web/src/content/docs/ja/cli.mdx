codemate CLI は、引数なしで実行すると、デフォルトで [TUI](/docs/tui) を開始します。

```bash
codemate
```

ただし、このページに記載されているようにコマンドも受け入れます。これにより、codemate をプログラム的に操作できるようになります。

```bash
codemate run "Explain how closures work in JavaScript"
```

---

### tui

codemate ターミナルユーザーインターフェイスを開始します。

```bash
codemate [project]
```

#### フラグ

| フラグ                                   | ショート                                | 説明                                                       |
| ---------------------------------------- | --------------------------------------- | ---------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c`                                    | 最後のセッションを続行                                     |
| <nobr><code>{"--session"}</code></nobr>  |                                         | 続行時にセッションをフォーク (`-s` または `--fork` と併用) |
| <nobr><code>{"--continue"}</code></nobr> | <nobr><code>{"--session"}</code></nobr> | 続行するセッション ID                                      |
| <nobr><code>{"--prompt"}</code></nobr>   |                                         | 使用のプロンプト                                           |
| <nobr><code>{"--model"}</code></nobr>    | `-m`                                    | プロバイダー/モデルの形式で使用するモデル                  |
| <nobr><code>{"--agent"}</code></nobr>    |                                         | 使用するエージェント                                       |
| <nobr><code>{"--port"}</code></nobr>     |                                         | リッスンするポート                                         |
| <nobr><code>{"--hostname"}</code></nobr> |                                         | リッスンするホスト名                                       |

## コマンド

codemate CLI には次のコマンドもあります。

---

### agent

codemate のエージェントを管理します。

```bash
codemate agent [command]
```

---

### attach

`serve` または `web` コマンドを使用して起動された、すでに実行中の codemate バックエンドサーバーにターミナルを接続します。

```bash
codemate attach [url]
```

これにより、リモート codemate バックエンドで TUI を使用できるようになります。例えば：

```bash
# Start the backend server for web/mobile access
codemate web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
codemate attach http://10.20.30.40:4096
```

#### フラグ

| フラグ                                   | ショート | 説明                                                                              |
| ---------------------------------------- | -------- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |          | TUI を開始する作業ディレクトリ                                                    |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | 最後のセッションを続行                                                            |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | 続行するセッション ID                                                             |
| <nobr><code>{"--fork"}</code></nobr>     |          | 続行時にセッションをフォーク（`--continue` または `--session` と使用）            |
| <nobr><code>{"--password"}</code></nobr> | `-p`     | Basic 認証パスワード（デフォルトは `codemate_SERVER_PASSWORD`）                   |
| <nobr><code>{"--username"}</code></nobr> | `-u`     | Basic 認証ユーザー名（デフォルトは `codemate_SERVER_USERNAME` または `codemate`） |

このコマンドは、カスタムシステムプロンプトとツール構成を使用して新しいエージェントを作成する手順を示します。

### auth

プロバイダーの資格情報とログインを管理するコマンド。

```bash
codemate auth [command]
```

---

#### login

codemate は [Models.dev](https://models.dev) のプロバイダーリストを利用しているため、`codemate auth login` を使用して、使用したいプロバイダーの API キーを構成できます。これは `~/.local/share/codemate/auth.json` に保存されます。

```bash
codemate auth login
```

codemate が起動すると、認証情報ファイルからプロバイダーがロードされます。また、環境またはプロジェクト内の `.env` ファイルで定義されたキーがあるかどうかも確認します。

---

#### list

認証情報ファイルに保存されているすべての認証されたプロバイダーをリストします。

```bash
codemate auth list
```

またはショートバージョン。

```bash
codemate auth ls
```

---

#### logout

資格情報ファイルからプロバイダーをクリアすることで、プロバイダーからログアウトします。

```bash
codemate auth logout
```

### mcp

モデルコンテキストプロトコルサーバーを管理します。

```bash
codemate mcp [command]
```

---

#### add

MCP サーバーを構成に追加します。

```bash
codemate mcp add
```

このコマンドは、ローカルまたはリモートの MCP サーバーを追加する手順を示します。

---

#### list

構成されているすべての MCP サーバーとその接続ステータスをリストします。

```bash
codemate mcp list
```

または、短いバージョンを使用してください。

```bash
codemate mcp ls
```

---

#### auth

OAuth 対応の MCP サーバーで認証します。

```bash
codemate mcp auth [name]
```

サーバー名を指定しない場合は、利用可能な OAuth 対応サーバーから選択するように求められます。

OAuth 対応サーバーとその認証ステータスを一覧表示することもできます。

```bash
codemate mcp auth list
```

または、短いバージョンを使用してください。

```bash
codemate mcp auth ls
```

---

#### logout

MCP サーバーの OAuth 資格情報を削除します。

```bash
codemate mcp logout [name]
```

---

#### debug

MCP サーバーの OAuth 接続の問題をデバッグします。

```bash
codemate mcp debug <name>
```

### models

構成されたプロバイダーから利用可能なすべてのモデルをリストします。

```bash
codemate models [provider]
```

このコマンドは、構成されたプロバイダー全体で利用可能なすべてのモデルを `provider/model` の形式で表示します。

これは、[設定](/docs/config/) で使用する正確なモデル名を把握するのに役立ちます。

オプションでプロバイダー ID を渡して、そのプロバイダーによってモデルをフィルターできます。

```bash
codemate models anthropic
```

#### フラグ

| フラグ                                  | 説明                                                            |
| --------------------------------------- | --------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | models.dev からモデルキャッシュを更新します。                   |
| <nobr><code>{"--verbose"}</code></nobr> | より詳細なモデル出力を使用します (コストなどのメタデータを含む) |

`--refresh` フラグを使用して、キャッシュされたモデルリストを更新します。これは、新しいモデルがプロバイダーに追加され、それを codemate で確認したい場合に便利です。

```bash
codemate models --refresh
```

### run

プロンプトを直接渡して、非対話モードで codemate を実行します。

```bash
codemate run [message..]
```

これは、スクリプト作成、自動化、または完全な TUI を起動せずに迅速な回答が必要な場合に便利です。例えば。

```bash "codemate run"
codemate run Explain the use of context in Go
```

実行中の `codemate serve` インスタンスにアタッチして、実行ごとの MCP サーバーのコールドブート時間を回避することもできます。

```bash
# Start a headless server in one terminal
codemate serve

# In another terminal, run commands that attach to it
codemate run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### フラグ

| フラグ                                   | ショート | 説明                                                                                      |
| ---------------------------------------- | -------- | ----------------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |          | 実行するコマンド。引数には message を使用します。                                         |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | 最後のセッションを続行                                                                    |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | 続行するセッション ID                                                                     |
| <nobr><code>{"--fork"}</code></nobr>     |          | 続行時にセッションをフォーク (`--continue` または `--session` と併用)                     |
| <nobr><code>{"--share"}</code></nobr>    |          | セッションを共有する                                                                      |
| <nobr><code>{"--model"}</code></nobr>    | `-m`     | プロバイダー/モデルの形式で使用するモデル                                                 |
| <nobr><code>{"--agent"}</code></nobr>    |          | 使用するエージェント                                                                      |
| <nobr><code>{"--file"}</code></nobr>     | `-f`     | メッセージに添付するファイル                                                              |
| <nobr><code>{"--format"}</code></nobr>   |          | 形式: デフォルト (フォーマット済み) または json (生の JSON イベント)                      |
| <nobr><code>{"--title"}</code></nobr>    |          | セッションのタイトル (値が指定されていない場合は、切り詰められたプロンプトが使用されます) |
| <nobr><code>{"--attach"}</code></nobr>   |          | 実行中の codemate サーバー (http://localhost:4096 など) に接続します。                    |
| <nobr><code>{"--password"}</code></nobr> | `-p`     | Basic 認証パスワード（デフォルトは `codemate_SERVER_PASSWORD`）                           |
| <nobr><code>{"--username"}</code></nobr> | `-u`     | Basic 認証ユーザー名（デフォルトは `codemate_SERVER_USERNAME` または `codemate`）         |
| <nobr><code>{"--dir"}</code></nobr>      |          | 実行ディレクトリ、またはアタッチ時のリモートサーバー上のパス                              |
| <nobr><code>{"--variant"}</code></nobr>  |          | モデルバリアント（プロバイダー固有の推論レベル）                                          |
| <nobr><code>{"--thinking"}</code></nobr> |          | 思考ブロックを表示                                                                        |
| <nobr><code>{"--port"}</code></nobr>     |          | ローカルサーバーのポート (デフォルトはランダムポート)                                     |

### serve

API アクセスのためにヘッドレス codemate サーバーを起動します。完全な HTTP インターフェイスについては、[サーバーのドキュメント](/docs/server) を確認してください。

```bash
codemate serve
```

これにより、TUI インターフェイスを使用せずに codemate 機能への API アクセスを提供する HTTP サーバーが起動します。 `codemate_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `codemate`)。

#### フラグ

| フラグ                                   | 説明                                    |
| ---------------------------------------- | --------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート                      |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名                    |
| <nobr><code>{"--mdns"}</code></nobr>     | mDNS 検出を有効にする                   |
| <nobr><code>{"--cors"}</code></nobr>     | CORS を許可する追加のブラウザーオリジン |

### stats

codemate セッションのトークンの使用状況とコストの統計を表示します。

```bash
codemate stats
```

#### フラグ

| フラグ                                  | 説明                                                                                       |
| --------------------------------------- | ------------------------------------------------------------------------------------------ |
| <nobr><code>{"--days"}</code></nobr>    | 過去 N 日間の統計を表示 (全期間)                                                           |
| <nobr><code>{"--tools"}</code></nobr>   | 表示するツールの数 (すべて)                                                                |
| <nobr><code>{"--models"}</code></nobr>  | モデルの使用状況の内訳を表示 (デフォルトでは非表示)。上位 N 件を表示するには数値を渡します |
| <nobr><code>{"--project"}</code></nobr> | プロジェクトでフィルタリング (全プロジェクト、空文字列: 現在のプロジェクト)                |

### export

セッションデータを JSON としてエクスポートします。

```bash
codemate export [sessionID]
```

セッション ID を指定しない場合は、利用可能なセッションから選択するように求められます。

---

### import

JSON ファイルまたは codemate 共有 URL からセッションデータをインポートします。

```bash
codemate import <file>
```

ローカルファイルまたは codemate 共有 URL からインポートできます。

```bash
codemate import session.json
codemate import https://opncd.ai/s/abc123
```

### web

Web インターフェイスを使用してヘッドレス codemate サーバーを起動します。

```bash
codemate web
```

これにより、HTTP サーバーが起動し、Web ブラウザが開き、Web インターフェイスを通じて codemate にアクセスします。 `codemate_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `codemate`)。

#### フラグ

| フラグ                                   | 説明                                    |
| ---------------------------------------- | --------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート                      |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名                    |
| <nobr><code>{"--mdns"}</code></nobr>     | mDNS 検出を有効にする                   |
| <nobr><code>{"--cors"}</code></nobr>     | CORS を許可する追加のブラウザーオリジン |

| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート   |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名 |

### uninstall

codemate をアンインストールし、関連ファイルをすべて削除します。

```bash
codemate uninstall
```

#### フラグ

| フラグ                                      | ショート | 説明                                         |
| ------------------------------------------- | -------- | -------------------------------------------- |
| <nobr><code>{"--keep-config"}</code></nobr> | `-c`     | 構成ファイルを保持する                       |
| <nobr><code>{"--keep-data"}</code></nobr>   | `-d`     | セッションデータとスナップショットを保持する |
| <nobr><code>{"--dry-run"}</code></nobr>     |          | 削除せずに削除される内容を表示する           |
| <nobr><code>{"--force"}</code></nobr>       | `-f`     | 確認プロンプトをスキップする                 |

### upgrade

codemate を最新バージョンまたは特定のバージョンに更新します。

```bash
codemate upgrade [target]
```

最新バージョンにアップグレードするには。

```bash
codemate upgrade
```

特定のバージョンにアップグレードするには。

```bash
codemate upgrade v0.1.48
```

#### フラグ

| フラグ                                 | ショート | 説明                                                   |
| -------------------------------------- | -------- | ------------------------------------------------------ |
| <nobr><code>{"--method"}</code></nobr> | `-m`     | 使用されたインストール方法。curl, npm, pnpm, bun, brew |

## 環境変数

codemate は環境変数を使用して構成できます。

| 変数                                  | タイプ   | 説明                                                                |
| ------------------------------------- | -------- | ------------------------------------------------------------------- |
| `codemate_AUTO_SHARE`                 | ブール値 | セッションを自動的に共有する                                        |

| `codemate_DISABLE_PRUNE`              | ブール値 | 古いデータのプルーニングを無効にする                                |
| `codemate_DISABLE_TERMINAL_TITLE`     | ブール値 | ターミナルタイトルの自動更新を無効にする                            |
| `codemate_PERMISSION`                 | 文字列   | インライン化された json 権限設定                                    |
| `codemate_DISABLE_DEFAULT_PLUGINS`    | ブール値 | デフォルトのプラグインを無効にする                                  |
| `codemate_DISABLE_LSP_DOWNLOAD`       | ブール値 | LSP サーバーの自動ダウンロードを無効にする                          |
| `codemate_ENABLE_EXPERIMENTAL_MODELS` | ブール値 | 実験モデルを有効にする                                              |
| `codemate_DISABLE_AUTOCOMPACT`        | ブール値 | 自動コンテキスト圧縮を無効にする                                    |
| `codemate_DISABLE_CLAUDE_CODE`        | ブール値 | `.claude` からの読み取りを無効にする (プロンプト + スキル)          |
| `codemate_DISABLE_CLAUDE_CODE_PROMPT` | ブール値 | `~/.claude/CLAUDE.md` の読み取りを無効にする                        |
| `codemate_DISABLE_CLAUDE_CODE_SKILLS` | ブール値 | `.claude/skills` のロードを無効にする                               |
| `codemate_DISABLE_MODELS_FETCH`       | ブール値 | リモートソースからのモデルの取得を無効にする                        |
| `codemate_FAKE_VCS`                   | 文字列   | テスト目的の偽の VCS プロバイダー                                   |
| `codemate_CLIENT`                     | 文字列   | クライアント識別子 (デフォルトは `cli`)                             |
| `codemate_ENABLE_EXA`                 | ブール値 | Exa Web 検索ツールを有効にする                                      |
| `codemate_SERVER_PASSWORD`            | 文字列   | `serve`/`web` の基本認証を有効にする                                |
| `codemate_SERVER_USERNAME`            | 文字列   | 基本認証ユーザー名 (デフォルト `codemate`) をオーバーライドします。 |
| `codemate_MODELS_URL`                 | 文字列   | モデル設定を取得するためのカスタム URL                              |
