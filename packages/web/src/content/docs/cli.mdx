The Codemate CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
codemate
```

But it also accepts commands as documented on this page. This allows you to interact with Codemate programmatically.

```bash
codemate run "Explain how closures work in JavaScript"
```

---

### tui

Start the Codemate terminal user interface.

```bash
codemate [project]
```

#### Flags

| Flag         | Short | Description                                                             |
| ------------ | ----- | ----------------------------------------------------------------------- |
| `--continue` | `-c`  | Continue the last session                                               |
| `--session`  | `-s`  | Session ID to continue                                                  |
| `--fork`     |       | Fork the session when continuing (use with `--continue` or `--session`) |
| `--prompt`   |       | Prompt to use                                                           |
| `--model`    | `-m`  | Model to use in the form of provider/model                              |
| `--agent`    |       | Agent to use                                                            |
| `--port`     |       | Port to listen on                                                       |
| `--hostname` |       | Hostname to listen on                                                   |

## Commands

The Codemate CLI also has the following commands.

---

### agent

Manage agents for Codemate.

```bash
codemate agent [command]
```

---

### attach

Attach a terminal to an already running Codemate backend server started via `serve` or `web` commands.

```bash
codemate attach [url]
```

This allows using the TUI with a remote Codemate backend. For example:

```bash
# Start the backend server for web/mobile access
codemate web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
codemate attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
codemate agent create
```

This command will guide you through creating a new agent with a custom system prompt and permission configuration. Anything you don't allow is denied in the generated agent's frontmatter.

#### Flags

| Flag            | Description                                                                                                                                                                                                                |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--path`        | Directory to write the agent file to (defaults to global or `.codemate/agent` based on the prompt)                                                                                                                         |
| `--description` | What the agent should do                                                                                                                                                                                                   |
| `--mode`        | Agent mode: `all`, `primary`, or `subagent`                                                                                                                                                                                |
| `--permissions` | Comma-separated list of permissions to allow (default: all). Available: `bash`, `read`, `edit`, `glob`, `grep`, `webfetch`, `task`, `todowrite`, `websearch`, `lsp`, `skill`. Anything omitted is denied. Alias: `--tools` |
| `--model`, `-m` | Model to use, in `provider/model` format                                                                                                                                                                                   |

Passing all of `--path`, `--description`, `--mode`, and `--permissions` runs the command non-interactively.

---

#### list

List all available agents.

```bash
codemate agent list
```

### auth

Command to manage credentials and login for providers.

```bash
codemate auth [command]
```

---

#### login

Codemate is powered by the provider list at [Models.dev](https://models.dev), so you can use `codemate auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/codemate/auth.json`.

```bash
codemate auth login
```

When Codemate starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
codemate auth list
```

Or the short version.

```bash
codemate auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
codemate auth logout
```

### github

Manage the GitHub agent for repository automation.

```bash
codemate github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
codemate github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
codemate github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
codemate mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
codemate mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
codemate mcp list
```

Or use the short version.

```bash
codemate mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
codemate mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
codemate mcp auth list
```

Or use the short version.

```bash
codemate mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
codemate mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
codemate mcp debug <name>
```

### models

List all available models from configured providers.

```bash
codemate models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
codemate models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in Codemate.

```bash
codemate models --refresh
```

### run

Run codemate in non-interactive mode by passing a prompt directly.

```bash
codemate run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "codemate run"
codemate run Explain the use of context in Go
```

You can also attach to a running `codemate serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
codemate serve

# In another terminal, run commands that attach to it
codemate run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag                             | Short | Description                                                             |
| -------------------------------- | ----- | ----------------------------------------------------------------------- |
| `--command`                      |       | The command to run, use message for args                                |
| `--continue`                     | `-c`  | Continue the last session                                               |
| `--session`                      | `-s`  | Session ID to continue                                                  |
| `--fork`                         |       | Fork the session when continuing (use with `--continue` or `--session`) |
| `--share`                        |       | Share the session                                                       |
| `--model`                        | `-m`  | Model to use in the form of provider/model                              |
| `--agent`                        |       | Agent to use                                                            |
| `--file`                         | `-f`  | File(s) to attach to message                                            |
| `--format`                       |       | Format: default (formatted) or json (raw JSON events)                   |
| `--title`                        |       | Title for the session (uses truncated prompt if no value provided)      |
| `--attach`                       |       | Attach to a running codemate server (e.g., http://localhost:4096)       |
| `--port`                         |       | Port for the local server (defaults to random port)                     |
| `--dangerously-skip-permissions` |       | Auto-approve permissions that are not explicitly denied (dangerous!)    |

### serve

Start a headless Codemate server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
codemate serve
```

This starts an HTTP server that provides API access to codemate functionality without the TUI interface. Set `CODEMATE_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `codemate`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

### stats

Show token usage and cost statistics for your Codemate sessions.

```bash
codemate stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
codemate export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or Codemate share URL.

```bash
codemate import <file>
```

You can import from a local file or an Codemate share URL.

```bash
codemate import session.json
codemate import https://opncd.ai/s/abc123
```

---

### web

Start a headless Codemate server with a web interface.

```bash
codemate web
```

This starts an HTTP server and opens a web browser to access Codemate through a web interface. Set `CODEMATE_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `codemate`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
codemate acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall Codemate and remove all related files.

```bash
codemate uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates codemate to the latest version or a specific version.

```bash
codemate upgrade [target]
```

To upgrade to the latest version.

```bash
codemate upgrade
```

To upgrade to a specific version.

```bash
codemate upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

Codemate can be configured using environment variables.

| Variable                              | Type    | Description                                       |
| ------------------------------------- | ------- | ------------------------------------------------- |
| `CODEMATE_AUTO_SHARE`                 | boolean | Automatically share sessions                      |
| `CODEMATE_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows            |
| `CODEMATE_CONFIG`                     | string  | Path to config file                               |
| `CODEMATE_TUI_CONFIG`                 | string  | Path to TUI config file                           |
| `CODEMATE_CONFIG_DIR`                 | string  | Path to config directory                          |
| `CODEMATE_CONFIG_CONTENT`             | string  | Inline json config content                        |
| `CODEMATE_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                   |
| `CODEMATE_DISABLE_PRUNE`              | boolean | Disable pruning of old data                       |
| `CODEMATE_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates          |
| `CODEMATE_PERMISSION`                 | string  | Inlined json permissions config                   |
| `CODEMATE_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                           |
| `CODEMATE_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads            |
| `CODEMATE_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                        |
| `CODEMATE_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction              |
| `CODEMATE_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)  |
| `CODEMATE_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`             |
| `CODEMATE_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                  |
| `CODEMATE_DISABLE_MODELS_FETCH`       | boolean | Disable fetching models from remote sources       |
| `CODEMATE_DISABLE_MOUSE`              | boolean | Disable mouse capture in the TUI                  |
| `CODEMATE_FAKE_VCS`                   | string  | Fake VCS provider for testing purposes            |
| `CODEMATE_CLIENT`                     | string  | Client identifier (defaults to `cli`)             |
| `CODEMATE_ENABLE_EXA`                 | boolean | Enable Exa web search tools                       |
| `CODEMATE_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`               |
| `CODEMATE_SERVER_USERNAME`            | string  | Override basic auth username (default `codemate`) |
| `CODEMATE_MODELS_URL`                 | string  | Custom URL for fetching models configuration      |
